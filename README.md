# 위잇 단계별 목표 관리 플래너 (Funnel Planner)

퍼널 단계별 목표 역산 · DB/광고비 진단 · 일일 예산 · 월별 실적 관리 대시보드.
**더블클릭 인라인 수정** + **Firebase 실시간 공유** 지원.

👉 바로 쓰기: `index.html` 을 브라우저로 열면 됩니다. (공유 웹주소는 GitHub Pages 참고)

---

## ✨ 주요 기능

- **퍼널 목표**: 가능 DB → 찾기 → 합자 → 육 → 영 → 센확 단계별 목표 자동 역산
- **더블클릭 수정**: 화면의 `센확 목표 / DB / 광고비` 숫자를 더블클릭하면 바로 수정
- **일일 예산**: 캠페인 시작·마감일 기준으로 일일 광고비·일일 목표·진행 페이스 자동 계산
- **DB/광고비 진단**: DB 단가, 목표 달성 가능 여부, 추가 필요 예산 진단
- **월별 실적**: 월별 단계 실적 입력 → 전환율 자동 계산
- **실시간 공유**: 한 명이 수정하면 같은 보드를 연 모두에게 즉시 반영 (Firebase)

---

## 🔗 실시간 공유 켜기 (Firebase 설정)

설정을 비워두면 자동으로 **로컬 전용**(각자 브라우저에만 저장)으로 동작합니다.
여러 명이 같은 데이터를 함께 쓰려면 아래 5분 설정을 한 번만 하면 됩니다.

### 1. Firebase 프로젝트 만들기
1. https://console.firebase.google.com 접속 (3G 구글 계정으로 로그인)
2. **프로젝트 추가** → 이름 입력 (예: `weit-planner`) → 생성
3. 왼쪽 메뉴 **빌드 → Realtime Database** → **데이터베이스 만들기**
   - 위치: `싱가포르(asia-southeast1)` 등 선택
   - 보안 규칙: 우선 **테스트 모드**로 시작

### 2. 웹 앱 등록 후 설정값 복사
1. 프로젝트 개요 옆 **⚙️ → 프로젝트 설정**
2. 하단 **내 앱 → 웹(`</>`)** 추가 → 앱 닉네임 입력
3. 표시되는 `firebaseConfig` 값에서 아래 4개를 복사

### 3. `index.html` 상단에 붙여넣기
`index.html` 의 `FIREBASE_CONFIG` 부분을 채웁니다:

```js
const FIREBASE_CONFIG = {
  apiKey:      "AIzaSy....",
  authDomain:  "weit-planner.firebaseapp.com",
  databaseURL: "https://weit-planner-default-rtdb.firebaseio.com",
  projectId:   "weit-planner",
};
```

저장 후 새로고침하면 우상단 배지가 **● 실시간 공유** 로 바뀝니다.

### 4. (중요) 보안 규칙
테스트 모드는 30일 뒤 잠깁니다. 내부 팀끼리만 쓸 거면 Realtime Database **규칙** 탭에서:

```json
{ "rules": { ".read": true, ".write": true } }
```

> ⚠️ 이 규칙은 URL을 아는 사람은 누구나 읽기/쓰기가 가능합니다. 소규모 신뢰 그룹 내부용으로만 쓰세요. 외부 공개가 걱정되면 로그인(Auth) 기반 규칙으로 강화하세요.
